# Point Oracle

The Point Oracle is Checkpoint's cryptographic verification system that bridges offchain points data with onchain trading. It ensures the integrity and authenticity of point claims through cryptographic signatures and provides real-time verification for the trading ecosystem.

## How It Works

The Point Oracle operates as a decentralized verification layer that:

<CardGroup cols={2}>
  <Card title="Signature Verification" icon="certificate" color="#32cd32">
    Validates cryptographic signatures from point-issuing protocols to authenticate point claims
  </Card>
  <Card title="Real-time Updates" icon="clock" color="#32cd32">
    Continuously monitors and updates point balances across supported protocols
  </Card>
  <Card title="Settlement Triggers" icon="handshake" color="#32cd32">
    Automatically initiates settlement processes when TGE events are detected
  </Card>
  <Card title="crosschain Support" icon="network-wired" color="#32cd32">
    Enables verification and settlement across multiple blockchain networks
  </Card>
</CardGroup>

## Oracle Architecture

The Point Oracle consists of three core components:

### 1. Signature Verification

<Steps>
  <Step title="Point Claim Submission">
    Users submit point claims with cryptographic signatures from the issuing protocol
  </Step>
  <Step title="Signature Validation">
    Oracle verifies the signature against the protocol's public key
  </Step>
  <Step title="Balance Confirmation">
    Confirmed balances are recorded onchain for trading eligibility
  </Step>
</Steps>

### 2. Real-time Monitoring

The Oracle continuously monitors:
- Point balance changes across protocols
- Token Generation Events (TGE) announcements
- Protocol updates and parameter changes
- Crosschain settlement requirements

### 3. Settlement Automation

<AccordionGroup>
  <Accordion title="TGE Detection" icon="bell" defaultOpen>
    Automatically detects when protocols announce their Token Generation Events and triggers settlement processes
  </Accordion>

  <Accordion title="Balance Reconciliation" icon="calculator">
    Reconciles final token distributions with traded point amounts to ensure accurate settlement
  </Accordion>

  <Accordion title="Crosschain Distribution" icon="globe">
    Coordinates token distribution across multiple chains using LayerZero infrastructure
  </Accordion>
</AccordionGroup>

### Verification Process

The Oracle follows a rigorous verification process:

1. **Signature Validation**: Verifies cryptographic signatures against protocol public keys
2. **Balance Verification**: Cross-references claimed balances with protocol data
3. **Temporal Validation**: Ensures signatures are within valid time windows
4. **Replay Protection**: Prevents reuse of previously verified signatures

## API Integration

Developers can integrate with the Point Oracle through our comprehensive API:

<CodeGroup>
```bash curl
curl -X GET "https://api.checkpoint.exchange/oracle/verify" \
  -H "Content-Type: application/json" \
  -d '{"address": "0x...", "protocol": "eigenlayer", "signature": "0x..."}'
```
```javascript JavaScript
const response = await fetch('https://api.checkpoint.exchange/oracle/verify', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    address: '0x...',
    protocol: 'eigenlayer',
    signature: '0x...'
  })
});
const verification = await response.json();
```
```python Python
import requests

response = requests.post('https://api.checkpoint.exchange/oracle/verify', 
  json={
    'address': '0x...',
    'protocol': 'eigenlayer', 
    'signature': '0x...'
  }
)
verification = response.json()
```
</CodeGroup>

## Oracle Economics

### Fee Structure

The Oracle operates on a fee-based model:

- **Verification Fees**: Small fees for signature verification requests
- **Settlement Fees**: Percentage-based fees on successful settlements
- **Crosschain Fees**: Additional fees for crosschain operations
